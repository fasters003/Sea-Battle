Инструкция по разработке игры «Морской бой» на C++ и SFML  

1. Введение

Цель данной работы — создать полноценную игру «Морской бой» с графическим интерфейсом. В проекте необходимо реализовать возможность ручной и автоматической расстановки кораблей, три уровня сложности искусственного интеллекта (бота), систему тактических способностей и сохранение персональной статистики игроков.

Необходимые инструменты:  

Среда разработки: Microsoft Visual Studio 2022.  

Графическая библиотека: SFML 2.5.1 (модули Graphics, Window, System).  

Дополнительные ресурсы: шрифт arial.ttf для корректного отображения текста.

2. Основные разделы

2.1. Подготовка среды разработки


Правильная настройка проекта — залог успешной сборки.  

Установите библиотеку SFML и распакуйте её на диск (например, в C:\SFML).  

В настройках проекта Visual Studio (Project -> Properties) выберите нужную архитектуру (рекомендуется x64).  

В C/C++ -> General -> Additional Include Directories укажите путь к папке include библиотеки SFML.  

В Linker -> General -> Additional Library Directories укажите путь к папке lib.  

В Linker -> Input -> Additional Dependencies пропишите: sfml-graphics.lib, sfml-window.lib, sfml-system.lib.  

Важно: Скопируйте файлы sfml-graphics-2.dll, sfml-window-2.dll, sfml-system-2.dll и файл шрифта arial.ttf в папку, где будет лежать ваш скомпилированный .exe файл.  


2.2. Разработка архитектуры и класса Cell


Игровое поле состоит из клеток. Создайте класс Cell, который будет хранить состояние отдельной ячейки.  

Используйте перечисление CellStatus со значениями: Empty (пусто), Ship (корабль), Miss (промах), Hit (попадание).  

Добавьте булевы флаги: hasMine (задел на будущее) и isIlluminated.  

Зачем нужен isIlluminated? Он используется для системы способностей. Когда игрок применяет «Радар» или «Прожектор», клетки меняют этот флаг на true, и метод отрисовки временно подсвечивает их желтым или голубым цветом, раскрывая туман войны.  


2.3. Реализация класса Ship


Класс Ship отвечает за логику отдельного судна.


Создайте структуру ShipPart с полями x, y и булевым флагом isHit для отслеживания состояния каждой палубы.  

В классе Ship объявите поля: size (размер), isHorizontal (ориентация) и вектор std::vector<ShipPart> parts.  

Реализуйте метод isSunk(). Он должен проходить циклом по всем элементам вектора parts и возвращать true только в том случае, если флаг isHit равен true у каждой палубы.  



2.4. Реализация класса Field (основная логика)  

Класс Field управляет двумерным массивом клеток 10×10 (std::vector<std::vector<Cell>> cells) и списком кораблей (std::vector<Ship> ships).  

Основные методы:  

clearField() — обнуляет матрицу клеток и очищает вектор кораблей.  

canPlaceShip(x, y, size, horizontal) — важнейший метод. Проверяет, не выходит ли корабль за границы поля и нет ли рядом других кораблей. Проверка должна охватывать не только сами клетки корабля, но и соседние с ними (от -1 до +1 по X и Y), чтобы исключить касание по диагонали.  

placeShipManually() / placeShipsRandomly() — методы для добавления кораблей в массив после успешной проверки canPlaceShip().  

undoLastShip() — удаляет последний добавленный корабль (через ships.pop_back()) и очищает его клетки на поле.  

saveState() / restoreState() — копируют текущие cells и ships в резервные переменные. Это нужно для кнопки «Вернуть», если игроку не понравилась автоматическая случайная расстановка.  

handleInputForBot(x, y) — обрабатывает выстрел по координатам. Возвращает -1 при промахе, 0 при попадании и размер корабля (например, 4), если попадание оказалось фатальным (корабль потоплен).  

draw() — отрисовывает поле. В качестве аргумента передаётся флаг isEnemy. Если он true, целые корабли скрываются цветом воды. Потопленные корабли (isSunk() == true) перечеркиваются двумя линиями (крестом) для наглядности.  

2.5. Реализация трёх уровней сложности бота


Интеллект противника реализуется в методе handleSmartBotInput(BotDifficulty diff):  

Easy (Лёгкий): Бот генерирует случайные координаты X и Y и стреляет, пока не попадёт в нетронутую клетку.  

Hard (Сложный): Бот анализирует поле. Если находит клетку со статусом Hit (подбитый, но не потопленный корабль), он включает режим добивания. С помощью массивов смещений dx и dy он проверяет соседей (вверх, вниз, влево, вправо) и бьёт туда. Если корабль потоплен, флаг добивания сбрасывается.  

Ultra (Ультра): Бот использует алгоритм «шахматной доски». Он стреляет только по клеткам, где сумма координат чётная ((i + j) % 2 == 0). Это позволяет найти любой корабль в два раза быстрее. При попадании логика работает как на Hard уровне.  

Флаги отслеживания: Используйте переменные trackingShip, lastHitX, lastHitY, чтобы бот «запоминал», где он ранил ваш корабль.  


2.6. Создание системы способностей


Используйте переменную abilityPoints (AP) для маны. За каждые 2 уничтоженных корабля противника начисляйте игроку +1 AP.  

Радар (2 AP): Метод useRadar(). Вычисляет координаты клика и устанавливает флаг isIlluminated = true для квадрата 2×2 на поле врага.  

Прожектор (1 AP): Метод useSpotlight(). При клике проверяет, есть ли под клеткой корабль. Если да, проходит по всем parts этого корабля и подсвечивает их.  

Двойной выстрел (2 AP): Задаёт счетчик doubleShotsRemaining = 2, позволяя игроку игнорировать передачу хода один раз.  


2.7. Разработка класса Button  

Так как в SFML нет готовых кнопок GUI, создайте свой класс Button.  

Поля: прямоугольник sf::RectangleShape shape, текст sf::Text text и шрифт sf::Font font.  

Текст необходимо отцентрировать. Для этого получите границы текста через getLocalBounds() и сместите origin: text.setOrigin(width / 2.0f, height / 2.0f). Затем поместите текст в центр прямоугольника.  

Метод isClicked(sf::Vector2i mousePos) проверяет нажатие: shape.getGlobalBounds().contains(static_cast<sf::Vector2f>(mousePos)).  



2.8. Реализация игрового цикла и состояний (main.cpp)


Главный цикл while (window.isOpen()) управляется машиной состояний (enum GameState):  


NameInput: Окно ввода имени. Мигание курсора (символа |) реализуется с помощью sf::Clock, который переключает флаг каждые 500 мс.  

Menu: Выбор сложности (Easy, Hard, Ultra) и переход к статистике.  

ShipPlacement: Игрок расставляет корабли. Левый клик — поставить, R — повернуть, Backspace — отменить последнюю установку, кнопки 1–4 для выбора размера.  

Playing: Сам бой. Игрок стреляет мышкой. Если промах — ход переходит боту. Ход бота осуществляется с задержкой 500 мс через таймер sf::Clock, чтобы игрок успевал осознать происходящее.  

GameOver: Конец игры. Показ статистики (время, счёт) и кнопка возврата в меню. 

StatsView: Отображение окна с историей боёв из файла.  


2.9. Сохранение статистики в файл


Реализуйте запись и чтение файла stats.txt.  

saveMatchToHistory(): Записывайте данные в формате Имя Сложность Результат Уничтожено Потеряно Время_в_секундах. Важно: перед записью замените все пробелы в имени игрока на нижние подчеркивания _, иначе чтение через std::ifstream >> сломается!  

getMatchHistoryString(): Читает файл построчно. Если записей больше 15, пропускайте первые, чтобы отобразить только последние 15 игр.

3. Примеры фрагментов кода



Пример 1 — Строгая проверка размещения корабля (с учётом диагоналей):  

```
bool Field::canPlaceShip(int x, int y, int size, bool horizontal) {
    for (int i = 0; i < size; ++i) {
        int cx = horizontal ? x + i : x;
        int cy = horizontal ? y : y + i;
        if (cx >= 10 || cy >= 10) return false;
        
        // Проверка самой клетки и всех 8 соседей вокруг неё
        for (int dy = -1; dy <= 1; ++dy) {
            for (int dx = -1; dx <= 1; ++dx) {
                int nx = cx + dx, ny = cy + dy;
                if (nx >= 0 && nx < 10 && ny >= 0 && ny < 10 &&
                    cells[ny][nx].status == CellStatus::Ship)
                    return false;
            }
        }
    }
    return true;
}
```


Пример 2 — Режим "добивания" для сложного бота:  

```
// Ищем подбитые палубы и стреляем вокруг них (вверх, вниз, влево, вправо)
int dx[] = { 0, 0, 1, -1 };
int dy[] = { 1, -1, 0, 0 };

for (int d = 0; d < 4; ++d) {
    int nx = j + dx[d], ny = i + dy[d];
    
    // Если соседняя клетка не промах и не подбита - стреляем в неё
    if (nx >= 0 && nx < 10 && ny >= 0 && ny < 10 &&
        cells[ny][nx].status != CellStatus::Miss &&
        cells[ny][nx].status != CellStatus::Hit) {
        
        // Предотвращаем стрельбу по диагонали при добивании
        if (horiz && dx[d] == 0) continue;
        if (vert && dy[d] == 0) continue;
        
        huntTargets.push_back({ nx, ny });
    }
}
```


Пример 3 — Реализация функции "Отмена случайной расстановки":

void Field::saveState() {

4. Заключение (Важные моменты)

При разработке обязательно обратите внимание на следующие критические точки:


Диагональные соседи: При проверке canPlaceShip корабли не должны касаться даже углами. Вложенные циклы от -1 до 1 решают эту задачу.  

Сброс таргетинга: Для сложного бота крайне важно правильно сбрасывать флаги trackingShip или очищать массив целей сразу после того, как корабль помечается как потопленный (isSunk() == true).  

Отрисовка крестов: При отрисовке потопленных кораблей используйте два пересекающихся sf::RectangleShape, выровняв их центры (через setOrigin) и повернув на 45 и -45 градусов через метод setRotation().  

Скорость игры: Обязательно используйте sf::Clock для задержки хода бота. Без неё бот расстреляет всю вашу эскадру за долю секунды, и вы даже не поймёте, что произошло.  

Парсинг истории: При сохранении имени игрока в файл заменяйте пробелы на символы _. Иначе стандартный оператор чтения in >> u8Name >> diff >> res сломается, считав часть имени за сложность.  

Сборка релиза: Не забудьте скопировать файл arial.ttf и все .dll файлы SFML в итоговую папку Release с вашим .exe, иначе при запуске на другом компьютере игра выдаст ошибку отсутствия библиотек.
